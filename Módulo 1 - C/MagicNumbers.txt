MAGIC NUMBERS

    ALGUNS DOS PROGRAMAS QUE ESCREVEMOS NO CS50 ATÉ AGORA TIVERAM ALGUNS NÚMEROS ESTRANHOS INCLUÍDOS NELES.

    POR EXEMPLO, NO PROBLEMA DO MARIO, LIMITAMOS A ALTURA DA PIRÂMIDE A 23. MAS O QUE 23 SIGNIFICA? A RAZÃO 
    POR LIMITARMOS É POR QUE A ALTURA PADRÃO DE UMA JANELA DO TERMINAL É 24. MAS QUAL O SIGNIFICADO PARA O
    CONTEXTO? O SIGNIFICADO DE 23 É IMEDIATAMENTE ÓBVIO PARA ALGUÉM QUE OLHA O PROGRAMA E TALVEZ TENHA UMA
    JANELA DE TERMINAL DE TAMANHO DIFERENTE? PROVAVELMENTE NÃO.

    EM GERAL, É UM PÉSSIMO HÁBITO ESCREVER CONSTANTES EM SEU CÓDIGO. AO FAZER, ISSO É CHAMADO DE NÚMERO 
    MÁGICO, ALGO QUE GERALMENTE QUEREMOS EVITAR.

    POR EXEMPLO:

        card deal_cards(deck name)
        {
            for (int i = 0; i < 52; i++)
            {
                // deal the card
            }
        }

    O NÚMERO MÁGICO QUE QUEREMOS EVITAR, É O 52.
    PARA EVITA-LO, DECLARAMOS ELE EM UMA VARIÁVEL EXTERNA:

        card deal_cards(deck name)
        {
            int tamanho_do_deck = 52;
            for (int i = 0; i < tamanho_do_deck; i++)
            {
                // deal the card
            }
        }
    
    ISSO TAMBÉM DÁ UM SIGNIFICADO MAIS INTUITIVO QUANDO, MAIS TARDE, NO LOOP 'for', DIZEMOS QUE 'i' 
    É MENOR QUE O TAMANHO_DO_DECK.

    E ISSO RESOLVE NOSSO PROBLEMA, DANDO UM SIGNIFICADO SIMBÓLICO À CONSTANTE. MAS ISSO INTRODUZ UM OUTRO PROBLEMA.

    MESMO QUE ESSA VARIÁVEL SEJA DECLARADA GLOBALMENTE, E SE HOUVER OUTRA FUNÇÃO EM NOSSO CONJUNTO DE FUNÇÕES
    QUE LIDA COM A MANIPULAÇÃO DE CARTAS QUE INADVERTIDAMENTE ALTERA O TAMANHO DO BARALHO? ESPECIALMENTE SE 
    ESTIVERMOS LIDANDO COM UM CONJUNTO DE CARTAS ONDE É NECESSÁRIO EMBARALHAR TODO O BARALHO. SE O TAMANHO DO
    BARALHO FOR DIMINUINDO EM 1, NÃO ESTAREMOS EMBARALHANDO TODAS AS CARTAS, POSSIVELMENTE. E SE ESSE VALOR 
    TALVEZ PUDESSE SER EXPLORADO POR UM MAL ATOR.

    C FORNECE O QUE É CHAMADO DE DIRETIVA DE PRÉ-PROCESSADOR, QUE TAMBÉM É CHAMADA DE MACRO PARA CRIAR CONSTANTES
    SIMBÓLICAS. POR EXEMPLO: #INCLUDE, É UM EXEMPLO DE DIRETIVA DE MACRO.

    A FORMA DE CRIAR CONSTANTES SIMBÓLICAS OU DAR UM NOME A UMA CONSTANTE PARA QUE TENHA MAIS SIGNIFICADO, É A 
    SEGUINTE:

        #define NAME REPLACEMENT

    QUANDO SEU PROGRAMA É COMPILADO, O QUE REALMENTE ACONTECE É O COMPILADOR PASSAR PELO SEU CÓDIGO E SUBSTITUIR 
    CADA INSTÂNCIA DA PALAVRA 'NOME' PELO QUE VOCÊ COLOCOU COMO 'REPLACEMENT'.

    POR EXEMPLO, SE SEU CÓDIGO TIVER INCLINAÇÕES MATEMÁTICAS, E TIVER UM 3,14159265 PERDIDO PELO SEU CÓDIGO, 
    PROVAVELMENTE SABERÁ QUE ESTA FALANDO DE 'PI'. MAS TALVEZ PODEMOS DAR UM SIGNIFICADO MAIS SIMBÓLICO. E 
    PODEMOS, EM VEZ DISSO, DIZER:

        #define PI 3.14159265

    E ASSIM DECLARAMOS NOSSA VARIÁVEL CONSTANTE SIMBÓLICA, SEM PERIGO DE SER ALTERADA POR OUTRA FUNÇÃO OU ATOR.
    BASTA APENAS USARMOS 'PI' NO NOSSO CÓDIGO, QUE SUBSTITUIRÁ '3.14159265' E ASSIM POR DIANTE, PARA QUE VOCÊ
    NÃO PRECISE DIGITA-LO TODA VEZ ENQUANTO SEU PROGRAMA AINDA TEM A FUNCIONALIDADE QUE VOCÊ ESPERA, PORQUE VOCÊ
    ESTÁ TRABALHANDO COM MANIPULAÇÃO, MULTIPLICAÇÃO, DIVISÃO, SEJA LA O QUE FOR POR PI.

    ENTÃO A MELHOR SOLUÇÃO PARA NOSSO PROBLEMA ANTERIOR SERIA:

        #define TAMANHO_DO_DECK 52

        card deal_cards(deck name)
        {
            for (int i = 0; i < TAMANHO_DO_DECK; i++)
            {
                // deal the card
            }
        }
